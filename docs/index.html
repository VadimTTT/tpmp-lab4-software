📌 Project Name: Воздушный извозчик

Описание: Отряд грузовых вертолётов выполняет доставку грузов и людей в труднодоступном горном районе. Приложение ведёт учёт вертолётов, членов экипажа и выполненных рейсов. Для каждого рейса рассчитывается заработок экипажа (5% от обычного, 10% от спецрейса), контролируется остаток лётного ресурса. Доступ разграничен: командир авиаотряда управляет всеми данными, а члены экипажа могут просматривать информацию только о своём вертолёте и своих рейсах (*).

📖 Глоссарий

ТерминОписание
ВертолётЛетательный аппарат, используемый для перевозки грузов и людей
ЭкипажГруппа из трёх пилотов, постоянно закреплённых за одним вертолётом
РейсПолёт вертолёта по заданному маршруту
Обычный рейсСтандартный рейс без особых условий
СпецрейсРейс с особыми условиями (повышенная опасность, срочность)
Капитальный ремонтПолное восстановление вертолёта с обнулением/заменой ресурса
Лётный ресурсМаксимальное количество часов полёта до следующего капремонта
НалётФактическое количество часов, наработанное вертолётом с момента последнего ремонта

🎯 Функциональные требования

Актёры

  • Командир авиаотряда — полный доступ (CRUD, отчёты, статистика).
  • Член экипажа — только просмотр своих данных и данных своего вертолёта (*).

Основные варианты использования

  • UC1: Управление вертолётами (CRUD)
  • UC2: Управление экипажем (CRUD)
  • UC3: Управление рейсами (CRUD + проверка ресурса)
  • UC4: Просмотр налёта и остатка ресурса по каждому вертолёту
  • UC5: Рейсы вертолёта за указанный период (с массой груза и числом пассажиров)
  • UC6: Статистика по спецрейсам (кол-во, масса груза, заработано денег)
  • UC7: Статистика по обычным рейсам
  • UC8: Вертолёт с максимальным количеством рейсов (с экипажем и заработком)
  • UC9: Экипаж с максимальным заработком (с перечнем рейсов)
  • UC10: Просмотр своих рейсов для члена экипажа (*)

🗄️ Схема базы данных (SQLite)

Основные сущности: helicopters, crew, flights. Дополнительные таблицы: salary_results, app_users, app_config.

CREATE TABLE helicopters (
    helicopter_id INTEGER PRIMARY KEY,
    serial_number TEXT UNIQUE,
    model TEXT,
    manufacture_date TEXT,
    capacity REAL,
    last_overhaul TEXT,
    flight_resource INTEGER
);

CREATE TABLE crew (
    crew_id INTEGER PRIMARY KEY,
    personnel_number TEXT UNIQUE,
    full_name TEXT,
    position TEXT,
    experience INTEGER,
    address TEXT,
    birth_year INTEGER,
    helicopter_id INTEGER REFERENCES helicopters
);

CREATE TABLE flights (
    flight_id INTEGER PRIMARY KEY,
    flight_date TEXT,
    helicopter_id INTEGER REFERENCES helicopters,
    flight_code TEXT CHECK (flight_code IN ('обычный','спец')),
    cargo_mass REAL,
    passenger_count INTEGER,
    duration INTEGER,
    cost REAL
);

🛠️ Installation & Usage

Установка локально

  1. Клонировать репозиторий: git clone https://github.com/VadimTTT/tpmp-lab4-software.git
  2. Перейти в папку проекта: cd tpmp-lab4-software
  3. Установить SQLite, GCC, make (если не установлены).
  4. Выполнить сборку: make
  5. Запустить приложение: make run

Использование

После запуска открывается консольное меню, где командир может управлять вертолётами, экипажем и рейсами, а также просматривать аналитику (статистика, максимальные показатели). Члены экипажа при аутентификации видят только свой вертолёт и свои рейсы (* – будет реализовано на уровне запросов в финальной версии).
